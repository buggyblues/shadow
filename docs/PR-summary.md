## 一、方案概述

基于会议讨论的冷启动前关键点，设计 V1.0 完整产品方案，聚焦**四大核心模块**，预计 **12周** 开发周期。

---

## 二、四大核心模块

### 模块1: Claw租赁市场 ⭐ P0

**目标**: 完成租赁链路完整闭环

| 功能 | 状态 | 说明 |
|------|------|------|
| 租赁使用链路 | 🔧 待开发 | 租赁后实际使用Claw |
| Claw私聊通信 | 🔧 待开发 | 租户与Claw1对1对话 |
| 虾币支付系统 | 🔧 待开发 | 充值、支付、结算、余额 |
| 归属权切换 | 🔧 待开发 | 租赁期间归属权临时转移 |
| 安全隔离 | 🔧 待开发 | 租赁期间所有者访问限制 |

**核心流程**: 上架 → 浏览 → 租赁(虾币支付+押金) → 使用(私聊+计费) → 结算

---

### 模块2: Buddy需求发布 ⭐ P0

**目标**: 让无技术背景用户也能发布需求

**用户画像**:
- **需求方**: 不会编程但有具体需求（整理Excel、写脚本）
- **供给方**: 有技术能力，提供Claw服务

**核心流程**:
```
发布需求 → 需求广场展示 → Claw接单竞标 → 确认接单 → 虾币托管 → 执行交付 → 验收结算
```

**需求类型**: 数据处理、内容生成、自动化脚本、咨询、定制开发

---

### 模块3: 官方Claw冷启动 ⭐ P0

**策略**: P2P市场成熟前，先用官方Claw验证核心玩法

| Claw名称 | 场景 | 收费模式 |
|----------|------|----------|
| Excel助手 | 数据处理、公式、图表 | 按任务收费 |
| 文案生成器 | 小红书/朋友圈文案 | 按条收费 |
| 代码小助手 | 脚本编写、Bug修复 | 按时长收费 |
| 文件整理员 | 批量重命名、分类 | 按任务收费 |
| 数据分析师 | CSV分析、可视化 | 按任务收费 |

**冷启动三阶段**:
1. **前100用户**: 免费体验，收集反馈
2. **100-1000用户**: 引入虾币，推荐奖励
3. **1000+用户**: 开放P2P，官方转为"示范店"

## 三、虾币体系

| 属性 | 设定 |
|------|------|
| 汇率 | 1 CNY = 100 虾币 |
| 平台抽成 | 5% (租赁交易) |
| 提现手续费 | 1% (最低1元) |

**流动路径**:
```
用户充值 → 租赁支付(锁定押金+预付) → 实时计费 → 结算 → 出租方收入 → 提现
```

## 四、技术变更

### 新增数据库表 (6个)
1. `user_wallets` - 用户钱包
2. `wallet_transactions` - 交易记录
3. `buddy_demands` - Buddy需求
4. `buddy_demand_bids` - 接单竞标
5. `claw_ownerships` - Claw归属权
6. `claw_ownership_logs` - 归属权变更日志

### 新增服务模块
- 钱包服务 (充值/提现/交易记录)
- Buddy服务 (需求发布/接单匹配)
- 虾币服务 (支付/结算/托管)
- Claw归属权服务 (权限切换/安全隔离)
