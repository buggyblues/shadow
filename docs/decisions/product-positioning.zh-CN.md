# 产品定位决策文档

> 本文档记录当前产品定位和文档口径。历史上关于 Space 迁移、超级个体作为唯一主线、Agent 资产作为主叙事的内容已废弃。

---

## Q1: 虾豆是什么？

**决策：** 虾豆是 AI 互动社区平台。

成员围绕兴趣或共同目标聚集在服务器里。进入服务器后，成员先看到社区桌面：公告、互动组件、分享内容、社区应用、频道入口、工作区入口和 Buddy 入口可以被摆放在这里。成员从社区桌面开始使用，再通过频道交流、工作区协作和 Buddy 服务逐渐融入社区。

Buddy 是为社区 7/24 小时服务的 AI 搭子。它服务社区成员，理解当前服务器上下文，帮助成员答疑、整理信息、执行任务和维护社区服务。

---

## Q2: 核心对象怎么定义？

| 对象 | 产品定义 |
| --- | --- |
| 服务器 | 承载社区的产品对象。一个服务器通常对应一个围绕兴趣或共同目标聚集的社区。 |
| 社区桌面 | 成员进入服务器后看到的正式入口。管理员可以摆放公告、互动组件、分享内容、社区应用、频道入口、工作区入口和 Buddy 入口。 |
| 频道 | 成员交流、讨论和协作的主要场所。 |
| 工作区 | 保存文件、文档、图片、代码、研究摘要、Buddy 结果和其他协作产出。 |
| 社区应用 | 安装在服务器里的共享应用，可以承载工具、互动组件、内容展示、协作流程和后续交易入口。 |
| 分享内容 | 成员分享给社区的内容总称，可以是攻略、教程、文件、应用、创作成果或其他形式。它只是概念层面的统一说法，不对应独立数据模型；特定场景下再使用作品、文件、应用或商品等具体名称。 |
| Buddy | 为社区成员持续提供服务的 AI 搭子，可以从频道、社区桌面、社区应用或固定沟通入口触发。 |
| 云电脑 | Buddy 的云端运行环境，用于让公共社区服务更适合 7/24 在线和安全隔离。本机运行仍可存在，但不适合作为公共服务的默认方式。 |

---

## Q3: 社区桌面怎么工作？

**决策：** 社区桌面是服务器的大厅，不是成员的私人桌面。

新人进入服务器后，可以直接看到社区公告、互动组件、分享内容、社区应用、频道入口、工作区入口和 Buddy 入口。不同社区可以自行决定桌面上最重要的内容。

第一阶段规则：

- 桌面布局全社区统一。
- 桌面内容由服务器管理员摆放。
- 后续管理员可以任命其他成员共同管理桌面。
- 服务器管理员可以允许游客查看社区桌面；未加入服务器的游客只能看，加入后才能使用 Buddy 服务。
- 桌面负责展示、引导和提醒；成员仍会通过频道、工作区、社区应用和 Buddy 服务继续参与社区。

---

## Q4: 新人默认进入哪里？

**决策：** 新人默认进入官方社区桌面。

官方服务器可通过环境变量配置：

- Web 默认入口：`VITE_SHADOW_OFFICIAL_OS_SERVER`
- 服务端默认加入：`SHADOWOB_OFFICIAL_OS_SERVER`

当这两个值指向同一个服务器 slug 或 id 时，已登录的 visitor/member 会默认加入该服务器，并在登录后进入 `/app/os?server=<server>`。`/os` 是当前实现路径，不作为产品命名对外强调。

“游客状态”第一阶段指 visitor 账号，而不是完全匿名浏览。服务器管理员允许游客访问时，visitor 可以查看社区桌面，理解社区内容；使用 Buddy 服务需要先加入服务器。需要更高权限的能力仍按现有会员/邀请码机制控制。

---

## Q5: Buddy 的默认策略是什么？

**决策：** 第一阶段不做每个服务器独立的默认向导 Buddy。

后续计划提供一个统一的官方 Buddy，名称为“虾豆”。它默认加入所有社区，并基于当前服务器上下文回答问题，例如社区介绍、频道推荐、分享内容说明、应用使用方式和基础咨询。官方 Buddy 的特殊权限暂不作为当前定位重点。

在此之前，服务器可以自行添加 Buddy。Buddy 的产品表达统一为“为社区 7/24 小时服务的 AI 搭子”。

---

## Q6: Inbox 怎么处理？

**决策：** Inbox 暂时保留为实现层和技术设计概念，不放进产品主叙事。

产品层面应优先表达为“联系 Buddy”“向 Buddy 分配任务”“查看 Buddy 服务状态”等具体动作。技术层面，某个 Buddy 可以有固定入口，便于成员集中沟通和投递任务，避免在碎片化频道里找不到对应 Buddy。

---

## Q7: 经济系统的位置是什么？

**决策：** 经济系统不是当前核心定位。

成员先在服务器里交流、分享内容和协作。随着分享内容、社区应用和 Buddy 服务增长，虾币、交易、购买、打赏、租用和结算可以释放更多社区价值。

当前文案不应把虾豆首先描述成“AI 商业基建”“Agent 资产平台”或“Buddy 租赁平台”。这些只能作为中后期能力出现。

---

## Q8: 与 Discord 的差异是什么？

Discord 的中心是频道聊天。虾豆把服务器、社区桌面、频道、工作区、社区应用、分享内容和 Buddy 服务放在同一个社区里。

差异点：

- 服务器有社区桌面，管理员可以把公告、互动组件、分享内容、应用和 Buddy 入口摆到第一屏。
- 分享内容只是概念层面的统一说法，不新增独立数据模型。
- 工作区是核心对象，保存文件、协作产出和 Buddy 结果。
- 社区应用是共享对象，可以被摆放到社区桌面，也可以和频道、工作区、Buddy 配合使用。
- Buddy 是为社区 7/24 小时服务的 AI 搭子，公共服务更适合运行在云电脑上。

## Q9: 当前需要清理的方向

已废弃：

- 对外把 Server 改名为 Space。
- 以“超级个体”作为唯一目标用户。
- 把“Agent 资产”“AI 商业基建”“Buddy 租赁”作为第一屏主叙事。
- 新人首先创建或租用 Buddy 的新手路径。
- 在产品文档里强调 OS 模式、社区操作系统或 Inbox。

保留但降级：

- Buddy 租赁、店铺、虾币和交易系统，作为社区分享内容和服务成熟后的经济能力。
- 云电脑，作为 Buddy 的云端运行环境和公共社区服务的推荐运行方式。
- 开源、自部署、SDK、CLI、OAuth，作为开发者和生态能力。

## Q10: 近期产品优先级

1. 官方社区桌面：支持通过环境变量配置默认服务器，并让 visitor 默认加入官方服务器。
2. 社区统一桌面布局：管理员管理桌面内容，成员看到同一套大厅。
3. 桌面对象模型：公告、互动组件、分享内容、社区应用、频道入口、工作区入口和 Buddy 入口都能成为大厅对象。
4. Buddy 服务路径：成员能从频道、社区桌面、社区应用或固定入口联系 Buddy、咨询问题和分配任务。
5. 文案收敛：README、官网简介、产品文档统一到“AI 互动社区平台”。
