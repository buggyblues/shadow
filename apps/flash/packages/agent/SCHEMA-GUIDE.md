# Schema Quick Reference Guide

> **All skills producing JSON data MUST follow the structural constraints in this file.**
> The authoritative type definitions are in `packages/types/src/card.ts` (TypeScript + typia).
> Validation is handled at runtime by `packages/types/src/validate.ts`.

---

## 📋 Schema Source of Truth

| Schema | Source | Purpose |
|--------|--------|---------|
| Knowledge Card | `packages/types/src/card.ts` — `CardKind`, `CardKindMetaMap` | Cards produced by all skills |
| Fragment | Defined inline (see below) | Intermediate output from material-analysis |
| Outline | Defined inline (see below) | PPT outline produced by the analyze skill |
| Enrich Instruction | Defined inline (see below) | Field enrichment for existing cards |

## 🔑 Card Common Required Fields (shared by all kinds)

```
action    : "create" | "enrich"        ← every output must have this
id        : "card-001" | "research-deep-analysis-001" | "insp-001"
kind      : one of the kinds listed below
title     : string (≤30 chars)
summary   : string (≤150 chars, one sentence, must not carry data)
sourceId  : string | null
linkedCardIds : string[]               ← bidirectional references
tags      : string[] (1–5 items)
priority  : "high" | "medium" | "low"
autoGenerated : true
rating    : 1–5 (0.5 increments)
deckIds?  : string[]                   ← belonging Deck IDs (optional)
meta?     : object                     ← extended metadata (optional)
```

## ⚠️ Critical Data Consistency Constraints (violation = structural error)

### 1. chart card: series and categories must be the same length

```
✅ categories: ["Q1","Q2","Q3","Q4"]    → 4 items
   series[0].data: [14, 15, 17, 19]     → 4 items ✓

❌ categories: ["Q1","Q2","Q3","Q4"]    → 4 items
   series[0].data: [14, 15, 17]         → 3 items ✗ structural error!
```

### 2. table card: row key names must match columns.key

```
✅ columns: [{key:"name"}, {key:"age"}]
   rows: [{name:"Alice", age:30}]        ✓

❌ columns: [{key:"name"}, {key:"age"}]
   rows: [{company:"ACME", year:2024}]   ✗ key mismatch!
```

### 3. comparison card: values and subjects must be the same length

```
✅ subjects: ["A", "B"]                  → 2 items
   dimensions[0].values: ["Good", "Bad"] → 2 items ✓

❌ subjects: ["A", "B"]                  → 2 items
   dimensions[0].values: ["Good","Bad","Average"]  → 3 items ✗ length mismatch!
```

## 📊 Kind Upgrade Hard Rules (downgrading = failure)

| Quantitative Trigger | Must Use kind | Prohibited Downgrade |
|--------------------|--------------|---------------------|
| Data series with ≥3 time points | `chart` | ~~`data`~~ |
| ≥3 subjects × ≥2 dimensions | `table` | ~~`data` / `summary`~~ |
| ≥3 ordered steps | `process` | ~~`keypoint`~~ |
| Structured comparison of ≥2 subjects | `comparison` | ~~`argument`~~ |
| ≥4 time-ordered events | `timeline` | ~~`summary`~~ |
| Original text has row/column table structure | `table` | ~~`summary`~~ |

## 📦 Fragment Required Fields

```
id         : "frag-001"
materialId : string
kind       : "text"|"image"|"data"|"chart"|"table"|"quote"|"idea"|"code"|"audio"|"video"
title      : string
content    : string (raw extracted text)
data       : object | null (structured pre-extraction)
```

**Fragment `data` field pre-extraction map:**

| Fragment kind | data field content |
|--------------|-------------------|
| `data` | `{ metrics[], period?, benchmark? }` |
| `chart` | `{ chartType, categories[], series[] }` |
| `table` | `{ columns[], rows[] }` |
| `quote` | `{ text, author?, source? }` |
| `code` | `{ language, code, filename? }` |
| `image` | `{ description, contentType?, labels[]? }` |
| `text/idea/audio/video` | `null` |

## ✅ Pre-Output Checklist

Before producing any JSON data, confirm each item:

- [ ] Every card has an `action` field
- [ ] `kind` is one of the supported types listed above
- [ ] **All required specialized fields** for that kind are filled
- [ ] `chart`: `series[].data.length === categories.length`
- [ ] `table`: `rows` key names exactly match `columns[].key`
- [ ] `comparison`: `dimensions[].values.length === subjects.length`
- [ ] Did not downgrade ≥3 time-point data to `data` (should be `chart`)
- [ ] Did not downgrade ≥3×2 multi-dimensional data to `data`/`summary` (should be `table`)
- [ ] `summary` field contains only a one-sentence abstract, not detailed data
- [ ] Did not use non-existent `content` or `structured` fields
- [ ] JSON format is valid (array wrapper `[{...}, ...]`)

## 🚫 Common Data Deformation Errors

| Error Pattern | Correct Approach |
|--------------|----------------|
| chart series.data length doesn't match categories | Keep lengths strictly equal |
| table rows contain keys not defined in columns | Row keys must match columns[].key |
| comparison values array length doesn't match subjects | Keep lengths strictly equal |
| Putting trend data (≥3 time points) in `data.metrics` | Use `chart` with `series`+`categories` |
| Putting comparison table data in `summary.body` | Use `table` with `columns`+`rows` |
| Writing detailed analysis >150 chars in `summary` field | `summary` ≤150 chars; detailed content in specialized fields |
| Using `card.content` or `card.structured.xxx` | Specialized fields are at top level, e.g. `card.metrics` directly |
| `data` card's `metrics[].value` written as string with unit | value is a number, unit goes in `unit` field |
| `evidence[].type` using custom types | Only allowed: statistic/example/expert/trend/analogy |
| `chartType` using non-standard values (e.g. "line"/"bar") | Must use: barChart/lineChart/areaChart/pieChart |
