# Product

## Register

product

## Users

Shadow serves people who build, operate, share, and use AI Buddies inside social communities. Desktop users are often connecting local agent runtimes, managing Buddy identities, and moving between local tools and community chat.

## Product Purpose

Shadow gives Buddies durable identities, permissions, budgets, delivery history, and community presence. The desktop app bridges local runtimes such as Claude Code, Codex CLI, OpenCode, Hermes Agent, and OpenClaw into the Shadow community so users can talk to local capabilities as Buddies.

## Brand Personality

Helpful, social, and operational. The interface should feel friendly enough for consumer users, but precise enough that runtime state, connection state, and errors are easy to trust.

## Anti-references

Avoid admin-only density, decorative SaaS dashboards, vague "Agent" labels when the concrete runtime or Buddy name is available, hidden failure states, and workflows that send users away from the desktop app for basic connection setup.

## Design Principles

1. Name the real thing: show the Buddy name, runtime name, and connection state directly.
2. Keep local and cloud actions distinct: users should understand whether they are changing this computer, the community Buddy, or both.
3. Make failures visible: connection and deletion actions need clear progress, errors, and recovery paths.
4. Prefer task flow over decoration: settings screens should help users scan, connect, and fix issues quickly.
5. Support multiple Buddies and runtimes without visual crowding.

## Accessibility & Inclusion

Use accessible labels for icon-only actions, visible focus states, sufficient contrast for muted text, standard modal semantics, and reduced-motion-safe state transitions.
